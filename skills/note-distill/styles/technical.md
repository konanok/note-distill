# Style: technical（默认）

## 定位

面向**开发者技术沉淀**。目标读者是半年后的自己或同事——不在当时对话现场，需要理解不只是"怎么做"，更要理解"为什么"。

## 写作哲学

- 以**可复现性**为第一优先：读者应能照着笔记独立复现方案
- 以**原理**驱动叙述：先讲清楚机制，再给操作步骤
- 代码必须可直接执行，变量用 `<placeholder>` 标注

## 对模板的覆盖规则

无覆盖。完全遵循 `quick-template.md` 或 `depth-template.md` 的原始结构。

## 标题规范

- quick 模式：动词开头，描述操作（如「压缩最近 N 次 git commit」）
- deep 模式：名词短语，描述主题（如「NUMA 节点放置失败排查」）

## frontmatter

无额外字段。遵循 adapter 的 frontmatter 默认值。

## wikilinks

适量使用，3-8 个/篇，指向技术概念。

## 延伸方向（可选）

subagent 可酌情添加此小节，列出与本笔记相关的延伸主题或值得深入的方向：

- {可深入的方向或相关概念} — {一句话说明为什么值得探索}
- 最多 3 条

## 适用模式

quick / deep / auto 均可。
